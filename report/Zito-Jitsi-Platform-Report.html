Engagement Overview

Zito Media is a regional telecommunications provider in the United States building a Jitsi-based video conferencing platform as a free value-add service for residential cable subscribers. The project scope was to take over a partially completed deployment, diagnose all issues preventing functional video calls, fix them, and deliver a reproducible Ansible-managed production system.

I was brought on March 22, 2026. The platform was completely non-functional when I started. Conference rooms loaded in the browser but every user saw a black screen with no audio or video. I traced the failure through the full media pipeline and determined the root cause was not a single misconfiguration but four independent errors compounding to break every stage of the call setup.

What I Fixed

I identified and corrected six critical issues in a single sprint. Each one alone was enough to prevent video calls from functioning. All six were present simultaneously.

1. Fatal JavaScript Syntax Error in config.js

The file at /etc/jitsi/meet/jitsi-00.zitovoice.com-config.js contained a var config.jwt = {...} block above the main var config = {} declaration. Dot notation cannot be used in a var binding. The browser failed to parse the entire config file, so Jitsi Meet had no BOSH URL, no domain, no MUC address. Nothing downstream could initialize.

Before (broken) 
/* jitsi-meet config.js */

var config.jwt = {
    enableUserRolesBasedOnToken: true
};

var config = {
    hosts: {
        domain: 'jitsi-00.zitovoice.com',
        muc: 'conference.jitsi-00.zitovoice.com'
    },
    bosh: '//jitsi-00.zitovoice.com/http-bind',
};
After (fixed) 
/* jitsi-meet config.js */

var config = {
    enableUserRolesBasedOnToken: true,

    hosts: {
        domain: 'jitsi-00.zitovoice.com',
        muc: 'conference.jitsi-00.zitovoice.com'
    },
    bosh: '//jitsi-00.zitovoice.com/http-bind',
};

Prosody was set to jitsi-anonymous authentication with all token settings commented out. JWT tokens were being generated but Prosody was ignoring them entirely. Anyone could have joined any meeting without authentication.

6. NGINX Routing Loop and Prosody Module Errors

The NGINX stream multiplexer had the turn_backend upstream pointing to jitsi-00.zitovoice.com:5349 instead of 127.0.0.1:5349, creating a routing loop. I also removed the pubsub module from the VirtualHost scope (Prosody 13 requires it as a Component) and a polls module reference that had no corresponding file on disk.

NGINX fix 
# Before (routing loop)
upstream turn_backend {
    server jitsi-00.zitovoice.com:5349;
}

# After (correct)
upstream turn_backend {
    server 127.0.0.1:5349;
}

Open Items
Guest Domain (Derek's April 1 Question)

Derek reported that copying the invite link from a meeting does not include a token, which prevents other users from joining. This is expected behavior. Jitsi's invite button copies the room URL without the JWT because tokens are server-signed secrets and cannot be embedded in client-generated links.

The standard production solution for this is the guest domain pattern. A moderator opens the token URL to create the room. Guests join via the plain invite link and enter as participants without needing a token. The room only exists while a token-authenticated moderator is present.

Implementation required 
-- Prosody: Add guest VirtualHost
VirtualHost "guest.jitsi-00.zitovoice.com"
    authentication = "anonymous"
    c2s_require_encryption = false
    modules_enabled = {
        "bosh";
        "ping";
    }

// config.js: Add anonymous domain
hosts: {
    domain: 'jitsi-00.zitovoice.com',
    anonymousdomain: 'guest.jitsi-00.zitovoice.com',
    muc: 'conference.jitsi-00.zitovoice.com'
}

// jicofo.conf: Enable authentication gating
jicofo {
  authentication: {
    enabled: true
    type: XMPP
    login-url: "jitsi-00.zitovoice.com"
  }
}

Systemd Startup Ordering

The startup race condition was fixed with a manual ordered restart. A server reboot would re-trigger the problem because systemd has no dependency ordering configured between the services. I will add systemd drop-in overrides to enforce the correct sequence.

UDP 10000 Investigation

The nftables counter for JVB's media port shows zero inbound packets. This is a significant concern. It means either the Incus host firewall or an upstream router is blocking inbound UDP on port 10000. If this port is unreachable, all media traffic is forced through TURN relay, which will not scale to hundreds of concurrent meetings.
